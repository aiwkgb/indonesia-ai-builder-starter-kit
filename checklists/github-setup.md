# GitHub Setup Checklist

Checklist ini membantu pemula menyiapkan repository GitHub sebelum project dibagikan atau dilanjutkan.

## Istilah singkat

- Repository: tempat menyimpan file project di GitHub.
- Commit: catatan perubahan yang kamu simpan.
- Push: mengirim perubahan dari komputer ke GitHub.
- Issue: tempat mencatat tugas, ide, bug, atau pertanyaan.
- Pull Request: usulan perubahan sebelum digabungkan ke project utama.

## 1. Repository basics

- [ ] Nama repository jelas
- [ ] Repository dibuat public jika memang ingin open-source
- [ ] Description menjelaskan project ini tentang apa
- [ ] Main branch sudah siap digunakan

## 2. Required files

- [ ] `README.md` tersedia
- [ ] `LICENSE` tersedia
- [ ] `CONTRIBUTING.md` tersedia jika kontribusi diharapkan
- [ ] `CODE_OF_CONDUCT.md` tersedia jika kontribusi komunitas diharapkan
- [ ] `.gitignore` tersedia jika project menggunakan file kode

## 3. Documentation links

- [ ] Link di README berfungsi
- [ ] Dokumen penting mudah ditemukan
- [ ] Template, checklist, dan example ditautkan jika relevan

## 4. Issue setup

- [ ] Issue template tersedia
- [ ] Roadmap atau ide improvement dicatat sebagai issue
- [ ] Label digunakan hanya jika membantu

## 5. Before publishing or sharing

- [ ] Tidak ada data privat atau secret yang ikut masuk
- [ ] Nama file mudah dipahami
- [ ] Status project dijelaskan dengan jujur
- [ ] Repo tidak mengklaim adopsi, user, star, atau download tanpa bukti

## Final check

- [ ] Apakah pemula bisa memahami repo ini dalam 5 menit?
- [ ] Apakah orang lain bisa menemukan harus mulai dari mana?
- [ ] Apakah repo siap menerima feedback atau kontribusi kecil?
