# Project Review Checklist

Checklist ini membantu pemula Indonesia meninjau project kecil yang dibuat dengan bantuan AI sebelum dibagikan, dipublikasikan, atau diminta feedback.

## 1. Project purpose

- [ ] Tujuan project jelas
- [ ] Target user jelas
- [ ] Masalah yang ingin diselesaikan dijelaskan dengan sederhana

## 2. Documentation

- [ ] README menjelaskan project ini tentang apa
- [ ] Langkah setup atau cara penggunaan mudah diikuti
- [ ] Link penting tidak rusak
- [ ] Screenshot atau contoh ditambahkan jika membantu

## 3. User experience

- [ ] Alur utama mudah dipahami
- [ ] Teks mudah dibaca
- [ ] Empty state atau error state sudah dipikirkan
- [ ] Project tidak membingungkan untuk user pertama kali

## 4. AI-assisted workflow

- [ ] Prompt atau instruksi AI didokumentasikan jika berguna
- [ ] Kode atau konten dari AI sudah direview
- [ ] Data sensitif atau privat tidak ikut masuk
- [ ] Klaim sudah dicek dan tidak dilebih-lebihkan

## 5. GitHub readiness

- [ ] File tersusun rapi
- [ ] Commit message jelas
- [ ] Issue atau TODO penting dicatat
- [ ] License dan catatan kontribusi tersedia jika dibutuhkan

## 6. Publishing readiness

- [ ] Testing dasar sudah dilakukan
- [ ] Link halaman publik atau demo berfungsi jika tersedia
- [ ] Batasan project disebutkan
- [ ] Project jujur tentang tahap pengembangannya saat ini

## Final review

- [ ] Apakah pemula bisa memahami project ini dalam 5 menit?
- [ ] Apakah orang lain bisa memberi feedback tanpa terlalu banyak bertanya hal dasar?
- [ ] Apakah project siap dibagikan secara publik sebagai versi awal?
