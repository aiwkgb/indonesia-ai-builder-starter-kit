# Project Review Checklist

## Content

- [ ] Project goal is clear
- [ ] Target user is clear
- [ ] Main problem is explained
- [ ] Main benefit is explained
- [ ] Writing is simple and easy to understand

## Structure

- [ ] README exists
- [ ] Documentation exists
- [ ] Templates are easy to copy
- [ ] Examples are organized
- [ ] Next steps are clear

## Quality

- [ ] No broken internal links
- [ ] No missing important files
- [ ] Instructions are beginner-friendly
- [ ] Project can be improved by contributors
