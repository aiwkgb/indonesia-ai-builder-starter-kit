# PRD Template

Template ini membantu kamu menjelaskan ide produk atau project dengan jelas sebelum meminta bantuan AI seperti Claude, Codex, ChatGPT, atau Gemini. Isi dengan sederhana. Tidak perlu sempurna dari awal.

## English summary

This template helps beginner builders describe a product or project idea clearly before asking AI tools to help with planning, writing, coding, review, or documentation.

## What this template is for

Gunakan template ini untuk menulis gambaran awal tentang apa yang ingin kamu bangun, siapa penggunanya, masalah apa yang ingin diselesaikan, fitur apa yang penting, dan batasan apa yang belum perlu dikerjakan.

PRD ini bukan dokumen formal yang rumit. Tujuannya adalah membuat ide kamu cukup jelas supaya manusia dan AI bisa membantu dengan lebih terarah.

## Who should use it

Template ini cocok untuk:

- beginner AI builders
- students
- creators
- solo founders
- small business owners
- non-technical users yang ingin bekerja dengan AI tools

## When to use it

Gunakan template ini:

- sebelum membangun project
- sebelum meminta AI membuat kode
- sebelum meminta Codex atau Claude Code mengedit repo
- sebelum membuat GitHub issue
- sebelum meminta feedback dari orang lain

## How to use it

1. Isi setiap bagian dengan jujur sesuai kondisi project kamu.
2. Gunakan jawaban pendek dan jelas.
3. Ganti placeholder dalam tanda kurung siku seperti `[Isi nama project di sini]`.
4. Minta AI untuk meninjau PRD sebelum mulai membangun.
5. Revisi PRD berdasarkan feedback sebelum masuk ke coding atau produksi konten.

`[Isi nama project di sini]`

## Example mini PRD

### 1. Project name

`Catatan Pesanan Harian`

### 2. One-line summary

`Website sederhana untuk membantu pemilik usaha kecil mencatat pesanan harian dari pelanggan.`

### 3. Problem

Pemilik usaha kecil sering mencatat pesanan lewat chat atau buku tulis, sehingga data mudah tercecer dan sulit dicek ulang.

### 4. Target users

Pemilik usaha kecil yang memakai HP setiap hari dan ingin cara pencatatan yang lebih rapi tanpa sistem yang rumit.

### 5. User goals

1. Mencatat pesanan baru dengan cepat.
2. Melihat daftar pesanan hari ini.
3. Menandai pesanan yang sudah selesai.

### 6. Main features

1. Form tambah pesanan.
2. Daftar pesanan harian.
3. Status pesanan: baru, diproses, selesai.

### 7. Out of scope

1. Pembayaran online.
2. Login multi-user.
3. Integrasi ekspedisi.

### 8. User flow

1. Pengguna membuka halaman utama.
2. Pengguna mengisi nama pelanggan dan detail pesanan.
3. Sistem menambahkan pesanan ke daftar.
4. Pengguna mengubah status saat pesanan diproses.
5. Pengguna melihat pesanan yang sudah selesai.

### 9. Content or data needed

- Nama pelanggan.
- Detail pesanan.
- Status pesanan.
- Tanggal pesanan.

### 10. AI tools that may help

- ChatGPT untuk menyederhanakan copy tombol dan label form.
- Claude untuk meninjau PRD.
- Codex untuk membantu membuat halaman web sederhana.

### 11. Technical notes

- Platform: web app sederhana.
- Framework/tools: belum ditentukan.
- Hosting/deployment: bisa ditentukan setelah versi awal selesai.
- Batasan teknis: harus nyaman digunakan di HP.

### 12. Success criteria

1. Pengguna bisa menambah pesanan dalam kurang dari 1 menit.
2. Daftar pesanan mudah dibaca di layar HP.
3. Status pesanan bisa diubah tanpa langkah yang membingungkan.

### 13. Risks and assumptions

Risiko:

1. Pengguna mungkin tetap lebih nyaman memakai chat.
2. Form terlalu panjang bisa membuat pencatatan lambat.

Asumsi:

1. Pengguna bersedia mencoba pencatatan sederhana lewat web.
2. Pengguna tidak membutuhkan login pada versi awal.

### 14. Open questions

1. Apakah data perlu disimpan permanen?
2. Apakah perlu fitur export?
3. Apakah satu perangkat cukup untuk versi awal?

### 15. Next actions

1. Review PRD dengan AI.
2. Buat versi paling sederhana dari form dan daftar pesanan.
3. Uji dengan 1-2 pengguna.

## AI review prompt

Copy dan paste prompt ini ke AI tool:

```text
Tolong review PRD berikut sebagai partner yang kritis dan beginner-friendly.

Tujuan review:
1. Cek apakah tujuan project sudah jelas.
2. Identifikasi detail yang masih hilang atau membingungkan.
3. Sederhanakan wording jika terlalu rumit.
4. Tandai klaim yang terdengar berlebihan, terlalu yakin, atau overclaiming.
5. Sarankan next steps yang kecil, realistis, dan bisa dikerjakan.
6. Ingatkan bahwa semua output AI tetap perlu direview oleh manusia sebelum digunakan.

Jangan langsung membuat kode.
Fokus dulu pada memperjelas PRD.

PRD:
[Tempel PRD saya di sini]
```

## Human review checklist

Sebelum mulai membangun, cek hal berikut:

- [ ] Project bisa dijelaskan dalam 1 kalimat sederhana.
- [ ] Masalah pengguna sudah jelas.
- [ ] Target pengguna tidak terlalu luas.
- [ ] Fitur utama cukup kecil untuk versi awal.
- [ ] Bagian out of scope sudah jelas.
- [ ] User flow bisa diikuti langkah demi langkah.
- [ ] Konten atau data yang dibutuhkan sudah dicatat.
- [ ] Catatan teknis tidak memaksa tools yang belum diperlukan.
- [ ] Success criteria realistis dan bisa dicek.
- [ ] Risiko, asumsi, dan pertanyaan terbuka sudah ditulis.
- [ ] Tidak ada klaim berlebihan tentang hasil, adopsi, pengguna, atau pendapatan.
- [ ] Output dari AI sudah atau akan direview oleh manusia.

## Notes

- PRD tidak perlu sempurna. Yang penting cukup jelas untuk memulai diskusi.
- Mulai dari versi kecil yang bisa diuji.
- Jaga klaim tetap jujur dan realistis.
- Output AI harus dicek manual oleh manusia, terutama untuk kode, data, keamanan, dan keputusan penting.
- Jangan biarkan AI langsung membangun sebelum tujuan project jelas.
