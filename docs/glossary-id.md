# Glosarium AI Builder Indonesia

Glosarium ini berisi istilah-istilah dasar yang sering muncul saat membangun project AI-assisted menggunakan Claude, Codex, GitHub, Vercel, dan workflow modern.

Dokumen ini dibuat untuk pemula Indonesia yang ingin memahami istilah teknis tanpa harus langsung membaca dokumentasi yang terlalu kompleks.

## Cara menggunakan glossary ini

Gunakan dokumen ini saat kamu:

- Baru belajar GitHub, AI tools, atau deployment
- Bingung dengan istilah seperti PRD, commit, deploy, issue, atau pull request
- Ingin berkontribusi ke repo open-source
- Sedang mengisi template project
- Sedang mengikuti workflow Claude + Codex + GitHub

Kamu tidak harus menghafal semua istilah. Cukup baca istilah yang sedang kamu temui saat membangun project.

---

## A

### AI

AI adalah singkatan dari Artificial Intelligence atau kecerdasan buatan.

Dalam konteks repo ini, AI biasanya berarti alat bantu seperti Claude, Codex, ChatGPT, Gemini, atau tool lain yang bisa membantu menulis, merancang, mengecek, atau memperbaiki project.

Contoh:

> Menggunakan Claude untuk membuat PRD, lalu memakai Codex untuk membantu membuat file project.

### AI-assisted building

AI-assisted building adalah proses membangun project dengan bantuan AI.

AI tidak menggantikan builder sepenuhnya. Builder tetap menentukan tujuan, konteks, arah produk, dan keputusan akhir.

Contoh:

> Kamu menulis brief landing page, lalu meminta AI membantu membuat struktur halaman, copywriting, dan kode awal.

### AI workflow

AI workflow adalah alur kerja saat menggunakan AI dalam project.

Contoh sederhana:

1. Tulis brief
2. Buat PRD
3. Minta AI membuat rencana
4. Bangun project
5. Review hasil
6. Perbaiki
7. Commit ke GitHub
8. Deploy

### API

API adalah cara agar satu sistem bisa berkomunikasi dengan sistem lain.

Contoh:

> Website mengambil data produk dari database melalui API.

Untuk pemula, cukup pahami API sebagai “jembatan komunikasi antar aplikasi.”

---

## B

### Backend

Backend adalah bagian belakang dari aplikasi yang biasanya mengurus data, server, login, database, dan proses yang tidak terlihat langsung oleh user.

Contoh:

> Sistem checkout, penyimpanan user, atau database order.

### Beginner-friendly

Beginner-friendly berarti mudah dipahami oleh pemula.

Dalam repo ini, dokumentasi dan template sebaiknya:

- Menggunakan bahasa sederhana
- Tidak terlalu banyak jargon
- Memberikan contoh
- Menjelaskan langkah demi langkah

### Branch

Branch adalah cabang kerja di Git.

Biasanya branch dipakai agar perubahan tidak langsung mengganggu versi utama project.

Contoh:

> Membuat branch `add-glossary-id` untuk menambahkan file glossary.

### Brief

Brief adalah penjelasan ringkas tentang project.

Brief biasanya berisi:

- Tujuan project
- Target user
- Masalah yang ingin diselesaikan
- Gaya desain
- Fitur yang dibutuhkan
- Batasan project

Brief yang jelas membuat AI lebih mudah membantu.

### Bug

Bug adalah masalah atau kesalahan dalam project.

Contoh:

> Tombol tidak bisa diklik, halaman error, gambar tidak muncul, atau link rusak.

---

## C

### Changelog

Changelog adalah catatan perubahan project dari waktu ke waktu.

Contoh:

> v0.1.0 — menambahkan README, template, checklist, dan dokumentasi awal.

Changelog membantu contributor dan user melihat perkembangan project.

### Checklist

Checklist adalah daftar hal yang perlu dicek atau dikerjakan.

Contoh:

> Sebelum deploy, cek apakah project bisa jalan lokal, mobile responsive, dan tidak ada link rusak.

### Claude

Claude adalah AI assistant yang bisa membantu membuat rencana, PRD, dokumentasi, instruksi project, copywriting, dan analisis.

Dalam workflow repo ini, Claude cocok dipakai untuk tahap perencanaan dan perapihan konteks.

### Code of Conduct

Code of Conduct adalah aturan perilaku dalam project open-source.

Tujuannya agar komunitas tetap aman, ramah, dan tidak merendahkan pemula.

### Codex

Codex adalah coding agent yang bisa membantu membuat, membaca, mengubah, atau memperbaiki file dalam project.

Dalam workflow repo ini, Codex cocok dipakai untuk tahap implementasi teknis.

### Commit

Commit adalah catatan perubahan di Git.

Setiap commit sebaiknya punya pesan yang jelas.

Contoh commit message:

```text
 docs: add Indonesian AI builder glossary
```

### Component

Component adalah bagian kecil dari UI atau aplikasi yang bisa digunakan ulang.

Contoh:

- Navbar
- Button
- Card
- Footer
- Pricing section

### CONTRIBUTING.md

`CONTRIBUTING.md` adalah file yang menjelaskan cara orang lain bisa membantu project.

Biasanya berisi:

- Cara kontribusi
- Prinsip kontribusi
- Langkah membuat pull request
- Jenis kontribusi yang diterima

### Contributor

Contributor adalah orang yang ikut membantu project open-source.

Kontribusi tidak harus selalu kode. Bisa juga berupa dokumentasi, contoh, checklist, perbaikan typo, atau ide template baru.

### CTA

CTA adalah singkatan dari Call To Action.

CTA adalah ajakan agar user melakukan aksi tertentu.

Contoh:

- Beli sekarang
- Download template
- Mulai dari sini
- Baca panduan
- Fork repo ini

### Documentation / Docs

Documentation atau docs adalah panduan tertulis yang menjelaskan cara memakai, memahami, atau mengembangkan project.

Contoh:

- `docs/workflow-claude-codex-github.md`
- `docs/glossary-id.md`

### Example project

## F

### Fork

Fork adalah menyalin repo orang lain ke akun GitHub sendiri.

Biasanya fork dipakai saat seseorang ingin:

- Mempelajari project
- Mengubah project
- Mengusulkan kontribusi
- Membuat versi sendiri

### Frontend

Frontend adalah bagian aplikasi yang terlihat dan digunakan langsung oleh user.

Contoh:

- Halaman website
- Tombol
- Form
- Menu

Full-stack berarti mencakup frontend dan backend.

Git adalah alat untuk mencatat perubahan file dalam project.

Git membantu kita melihat:

- Bagaimana kembali ke versi sebelumnya

### GitHub Issues

GitHub Issues adalah fitur untuk mencatat tugas, bug, ide, atau roadmap.

Contoh issue:

Issue membantu project terlihat aktif, rapi, dan terbuka untuk kontribusi.

## Istilah yang sering tertukar

### Brief vs PRD

Brief biasanya lebih ringkas.

PRD lebih lengkap dan detail.

Gunakan brief untuk menjelaskan ide awal. Gunakan PRD saat project sudah ingin dibangun dengan lebih serius.

### Git vs GitHub

Git adalah alat untuk mencatat perubahan file.

GitHub adalah platform online untuk menyimpan repo dan kolaborasi.

### Commit vs Pull Request

Commit adalah satu catatan perubahan.

Pull Request adalah usulan untuk menggabungkan perubahan ke repo utama.

### Deploy vs Release

Deploy berarti mempublikasikan project agar bisa diakses online.

Release berarti menandai versi resmi project.

### UI vs UX

UI adalah tampilan.

UX adalah pengalaman saat menggunakan tampilan tersebut.
