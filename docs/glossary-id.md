# Glosarium AI Builder Indonesia

Glosarium ini berisi istilah-istilah dasar yang sering muncul saat membangun project AI-assisted menggunakan Claude, Codex, GitHub, Vercel, dan workflow modern.

Dokumen ini dibuat untuk pemula Indonesia yang ingin memahami istilah teknis tanpa harus langsung membaca dokumentasi yang terlalu kompleks.

> Catatan: Dokumen ini menggunakan Bahasa Indonesia sebagai bahasa utama. Ringkasan Bahasa Inggris disediakan untuk contributor internasional yang ingin memahami tujuan dokumen ini.

## English summary

- This glossary explains basic AI-assisted building terms for Indonesian beginners.
- It focuses on practical terms used in Claude, Codex, GitHub, Vercel, and modern project workflows.
- Users can read only the terms they currently need instead of memorizing everything.
- The goal is to reduce confusion for beginner builders who are new to technical documentation.
- Bahasa Indonesia remains the primary language of this glossary.

## Cara menggunakan glossary ini

Gunakan dokumen ini saat kamu:

- Baru belajar GitHub, AI tools, atau deployment
- Bingung dengan istilah seperti PRD, commit, deploy, issue, atau pull request
- Ingin berkontribusi ke repo open-source
- Sedang mengisi template project
- Sedang mengikuti workflow Claude + Codex + GitHub

Kamu tidak harus menghafal semua istilah. Cukup baca istilah yang sedang kamu temui saat membangun project.

## A

### AI

AI adalah singkatan dari Artificial Intelligence atau kecerdasan buatan.

Dalam konteks repo ini, AI biasanya berarti alat bantu seperti Claude, Codex, ChatGPT, Gemini, atau tool lain yang bisa membantu menulis, merancang, mengecek, atau memperbaiki project.

Contoh:

> Menggunakan Claude untuk membuat PRD, lalu memakai Codex untuk membantu membuat file project.

### AI-assisted building

AI-assisted building adalah proses membangun project dengan bantuan AI.

AI tidak menggantikan builder sepenuhnya. Builder tetap menentukan tujuan, konteks, arah produk, dan keputusan akhir.

Contoh:

> Kamu menulis brief landing page, lalu meminta AI membantu membuat struktur halaman, copywriting, dan kode awal.

### AI workflow

AI workflow adalah alur kerja saat menggunakan AI dalam project.

Contoh sederhana:

1. Tulis brief
2. Buat PRD
3. Minta AI membuat rencana
4. Bangun project
5. Review hasil
6. Perbaiki
7. Commit ke GitHub
8. Deploy

### API

API adalah cara agar satu sistem bisa berkomunikasi dengan sistem lain.

Contoh:

> Website mengambil data produk dari database melalui API.

Untuk pemula, cukup pahami API sebagai “jembatan komunikasi antar aplikasi.”

## B

### Backend

Backend adalah bagian belakang dari aplikasi yang biasanya mengurus data, server, login, database, dan proses yang tidak terlihat langsung oleh user.

Contoh:

> Sistem checkout, penyimpanan user, atau database order.
