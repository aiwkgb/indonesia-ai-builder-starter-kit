# Workflow: Claude + Codex + GitHub

Panduan ini memakai struktur bilingual ringan. Bahasa Inggris dipertahankan agar mudah dibaca oleh international contributors, sementara ringkasan Bahasa Indonesia membantu pemula Indonesia memahami alurnya dengan cepat.

## Ringkasan Bahasa Indonesia

- Workflow ini membantu pemula memakai AI tanpa kehilangan arah project.
- Gunakan Claude untuk merancang PRD, brand guideline, struktur project, draft tulisan, dan rencana implementasi.
- Gunakan Codex untuk membuat file, mengedit code, memperbaiki bug, refactor, menambah komponen, dan update dokumentasi.
- Simpan semua pekerjaan penting di GitHub agar project punya riwayat perubahan yang jelas.
- Deploy project ke Vercel agar hasilnya bisa dites dan direview secara publik.
- Setelah testing, buat issue dan perbaiki project secara bertahap.

## Overview

This workflow helps beginners use AI tools without losing project direction.

## Step 1: Plan with Claude

Use Claude to create:

- PRD
- Brand guideline
- Project structure
- Writing draft
- Implementation plan

## Step 2: Build or edit with Codex

Use Codex for:

- Creating files
- Editing code
- Fixing bugs
- Refactoring
- Adding components
- Updating documentation

## Step 3: Save everything in GitHub

GitHub is the source of truth.

Keep:

- Code
- Documentation
- Templates
- Changelog
- Issues
- Roadmap

## Step 4: Deploy with Vercel

Use Vercel for fast public previews so other people can test and review your work.

## Step 5: Iterate

After testing, create issues and improve the project step by step.
